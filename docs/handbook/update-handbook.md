# ハンドブックの更新方法

## 利用ソフトウェア

本ハンドブックは、[Docusaurus 3.7](https://docusaurus.io/blog/releases/3.7)を利用して作成している。
詳細な更新方法やAPI等は、[Docusaurus](https://docusaurus.io/)を参照する。

## 更新ルール

本ハンドブックは[ハンドブックファースト](./handbook-first)の理念に沿って運用するため、誰もが安心して加筆・修正できる状態を維持することが重要である。
そのため、更新時には以下のルールに従う。

### 更新フロー

すべての変更は Pull Request を通じて反映する。`main` ブランチへの直接 push は原則として行わない。

1. `main` から作業用ブランチを作成する
2. 変更をコミットし、GitHub に push する
3. `main` ブランチに対して Pull Request を作成する
4. [ハンドブックレビュー](./review)を受ける
5. 承認後にマージする

### 軽微な修正の扱い

以下のような軽微な修正についても、原則として Pull Request を経由して反映する。

- 誤字脱字の修正
- リンク切れの修正
- 体裁・フォーマットの調整

ただし、内容の意味を変えない修正に限り、レビューを簡略化してよい。
文意に影響する変更が含まれる場合は、通常のレビューを実施する。

### 書き方ガイドライン

書き方を統一することで、読み手が内容を受け取りやすくなる。そのため、以下の方針に従って記述する。

- 文体は「である調」で統一する
- ページのタイトルは H1 （`#`）で記述し、本文中の見出しは H2 （`##`）以降を用いる
- ページの並び順は frontmatter の `sidebar_position` で制御し、同じ階層内で重複させない
- 同じ内容を複数箇所に記載しない（[SSoT](./handbook-first/what-is-ssot)の原則に従う）
- 不確実な情報や未確定の内容は、下書きであることを明記する
